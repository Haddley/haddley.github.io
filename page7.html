pageContext

A SharePoint Web Part can do interesting things if it has "context".

In this example the ICalendarProps interface is used to pass values from the CalendarWebPart instance to the Calendar React component. So we add context of type WebPartContext to the ICalendarProps interface and  update the CalendarWebPart's render method to pass "this.context" to the Calendar component.

Running "gulp serve" allows us to test the result in the "Local Workbench" or a SharePoint Online site using the /_layouts/15/workbench.aspx page.

TypeScript and React

The generated CalendarWebPart class extends the Microsoft BaseClientSideWebPart class and that the generated Calendar class extends React.Component.

The generated CalendarWebPart class manages state and that the Calendar class has state passed to it via props.

Before doing additional work on the Calendar component we should reimplement it as a function. This (in my opinion) makes the Calendar component code much easier to read and if the Calendar component needs to maintain state in the future we can take advantage of React useState and useEffect hooks.
